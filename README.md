# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные
В приложении используются две основные сущности данных: товар и покупатель, которые описываются интерфейсами TypeScript и используются для типизации данных во всех слоях приложения.

### Товар (IProduct)
Интерфейс IProduct описывает товар, представленный в каталоге интернет-магазина

Поля:
`id: string` — уникальный идентификатор товара;
`title: string` — название товара;
`description: string` — описание товара;
`image: string` — ссылка на изображение товара;
`category: string` — категория товара;
`price: number | null` — цена товара. Может быть равна `null`, если товар недоступен для покупки

### Покупатель (IBuyer)
Интерфейс описывает данные покупателя, необходимые для оформления заказа.

Поля:
`payment: TPayment` — выбранный способ оплаты (`card` или `cash`);
`address: string` — адрес доставки;
`email: string` — электронная почта;
`phone: string` — номер телефона

### Данные заказа
Для взаимодействия с сервером используются дополнительные типы данных, которые описывают структуру запроса и ответа при оформлении заказа.

Типы данных заказа описываются в файле `src/types/index.ts` и используют существующие интерфейсы товара и покупателя

## Модели данных
Модели данных относятся к слою Model архитектуры MVP и отвечают за хранение и обработку данных приложения.
Они не зависят от DOM и не содержат логики отображения. Это позволяет использовать их независимо от интерфейса.
Модели не выполняют сетевые запросы и не взаимодействуют напрямую друг с другом. Каждая из моделей решает строго ограниченный набор задач, связанных с хранением и обработкой данных.

### Products
Класс `Products` предназначен для работы с каталогом товаров

Отвечает за:
хранение полного списка товаров;
хранение товара, выбранного пользователем для детального просмотра

Поля класса:
`items: IProduct[]` — массив всех товаров каталога;
`selected: IProduct | null` — поле selected, в котором хранится выбранный товар или null, если товар не выбран

Методы:
`setItems(items: IProduct[]): void` - сохраняет массив товаров;
`getItems(): IProduct[]` - возвращает массив товаров;
`getById(id: string): IProduct | undefined` - возвращает товар по идентификатору;
`setSelected(item: IProduct | null): void` - сохраняет выбранный товар;
`getSelected(): IProduct | null` - возвращает выбранный товар

### Cart
Класс Cart отвечает за хранение товаров, добавленных пользователем в корзину.

Отвечает за:
добавление и удаление товаров;
расчет общей стоимости и количества товаров;
проверка наличия товара в корзине

Поля класса:
`items: IProduct[]` - массив товаров в корзине.

Методы:
`getItems(): IProduct[]` - возвращает товары в корзине;
`add(item: IProduct): void` — добавляет товар в корзину;
`remove(item: IProduct): void` — удаляет товар из корзины;
`clear(): void` — очищает корзину;
`getTotal(): number` — возвращает общую стоимость товаров;
`getCount(): number` — возвращает количество товаров в корзине;
`hasById(id: string): boolean` — проверяет наличие товара по идентификатору

### Buyer
Класс `Buyer` отвечает за хранение и валидацию данных покупателя при оформлении заказа.

Отвечает за:
хранение данных покупателя;
частичное обновление данных;
валидация данных формы заказа

Поля класса:
`payment?: TPayment` — выбранный способ оплаты;
`address?: string` — адрес доставки;
`email?: string` — электронная почта;
`phone?: string` — телефон покупателя

Методы:
`setData(data: Partial<IBuyer>): void` — сохраняет переданные данные, не удаляя ранее сохранённые;
`getData(): IBuyer` — возвращает данные покупателя;
`clear(): void` — очищает все данные покупателя;
`validate(): Record<string, string>` — выполняет проверку данных и возвращает объект с ошибками валидации

## Слой коммуникации
Слой коммуникации отвечает за взаимодействие приложения с сервером и реализован в виде отдельного класса, использующего композицию с классом `Api`.

Класс принимает в конструктор объект, реализующий интерфейс `IApi`, и использует его методы для выполнения HTTP-запросов. Это позволяет изолировать работу с сетью от остальной логики приложения.

Слой коммуникации решает две основные задачи: получение каталога товаров с сервера и отправку данных заказа при оформлении покупки.

Методы класса:
`getProducts(): Promise<IProduct[]>` — выполняет GET-запрос на эндпоинт `/product/`;
`postOrder(data: OrderRequest): Promise<OrderResponse>` - выполняет POST-запрос
  на эндпоинт `/order/` и передаёт данные заказа

## Слой представления (View)
Слой View отвечает за отображение интерфейса приложения и обработку пользовательских действий.  
Каждый класс представления отвечает за один конкретный блок разметки.

### Component
Базовый  класс для всех компонентов представления.

Отвечает за:
хранение ссылки на корневой DOM-элемент;
обновление интерфейса через метод `render`

Методы:
`render(data?: Partial<T>): HTMLElement` — обновляет DOM-элемент через сеттеры и возвращает его;
`setImage(...)` — вспомогательный метод для установки изображений

### Header
Компонент шапки сайта.

Отвечает за:
отображение счётчика товаров в корзине;
генерацию события открытия корзины

Генерирует событие `basket:open`

### Gallery
Компонент каталога товаров на главной странице.

Отвечает за:
отображение списка карточек товаров;
обновление DOM при изменении каталога

### Карточки товаров
Все карточки товаров наследуются от класса `Card`.

#### Card
Базовый класс карточки товара.

Отвечает за:
отображение названия и цены товара;
обработку кликов по карточке и кнопке действия

#### CardCatalog
Карточка товара в каталоге.

Отвечает за:
отображение краткой информации о товаре;
генерацию события выбора товара.

Presenter в `main.ts` в обработчике `onClick` эмитит событие `card:select`

#### CardPreview
Карточка детального просмотра товара в модальном окне.

Отвечает за:
отображение полной информации о товаре;
управление кнопкой «Купить / Удалить из корзины»

Presenter в `main.ts` в обработчике `onButtonClick` эмитит событие `product:toggle`

#### CardBasket
Карточка товара в корзине.

Отвечает за:
отображение товара в списке корзины;
удаление товара из корзины

эмитит событие `basket:item:remove`

### Modal
Компонент модального окна.

Отвечает за:
открытие и закрытие модального окна;
отображение переданного контента

эмитит событие `modal:close`

### Basket
Компонент корзины.

Отвечает за:
отображение списка товаров;
отображение общей стоимости;
управление кнопкой оформления заказа

эмитит событие `order:open`

### Формы оформления заказа
Обе формы наследуются от абстрактного класса `Form`

#### OrderForm
Форма первого шага оформления заказа.

Отвечает за:
выбор способа оплаты;
ввод адреса доставки;
валидацию данных

эмитит события:
`order:change`
`order:submit`

#### ContactsForm
Форма второго шага оформления заказа.

Отвечает за:
ввод email и телефона;
валидацию данных

эмитит события:
`contacts:change`,
`contacts:submit`

### Success
Компонент успешного оформления заказа.

Отвечает за:
отображение итоговой суммы;
закрытие модального окна

эмитит событие `success:close`

## Презентер (Presenter)
Презентер реализован в файле `src/main.ts` и отвечает за логику работы приложения

#### События представлений:
`card:select`
`product:toggle`
`basket:open`
`basket:item:remove`
`order:open`
`order:change`
`order:submit`
`contacts:change`
`contacts:submit`
